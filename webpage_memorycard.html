Memorycard 앱 제작 동기 정보 시간에 다루어 본 앱인벤터를 이용해 실용적인 앱을 만들어 보고 싶어서 평상시 공부할 때도 사용할 수 있는 영단어 카드 암기 앱을 제작하기로 했다.

계획
사용자 로그인 정보를 저장하고, 사용자의 계정에 사용자가 만든 영단어 암기 카드 세트 또는 국어 서술형 카드 세트, 아니면 탐구 암기용 카드 세트를 만들 수 있다.
학습에서 카드 선택을 통해 선택할 수 있는 카드세트를 카드 학습하기, 뜻 맞추기의 두 가지 방법으로 학습하고 친구들과 공유 할 수 있는 앱을 만들고 싶었다.
추가적으로 로그인 화면에 오늘의 단어 탭을 만들어 단어 5개와 그 뜻을 랜덤으로 보여주는 기능을 만들고 싶었다.
Firebase는 구글이 운영하는 서비스로, 무료로 서버에 데이터베이스를 저장할 수 있는 서비스를 제공한다.
이 서비스를 활용해 사용자 정보와 영단어 카드 세트를 데이터로 저장하기로 계획했다.
다음으로 한영사전 open api를 활용해 오늘의 단어 탭을 만들기로 했다.
Firebase란 구글이 소유하고 있는 모바일 애플리케이션 개발 플랫폼으로, 데이터를 저장하고 변경할 수 있는 데이터베이스이다.
앱 인벤터에서 Firebase를 연결하면 앱을 닫아도 데이터가 사라지지 않게 할 수 있다.
이 프로젝트를 위해 만든 상위 분류의 데이터 그룹들:
_System_Accountinfo : 회원가입 시 회원 정보 데이터
_System_Friendinfo: 회원의 친구 정보 데이터
국어: 국어 암기 자료 저장
영어: 영어 암기 자료 저장
탐구: 탐구 암기 자료 저장
어려웠던 점 첫번째로 어려웠던 점은 무료 배포된 한영사전 open api를 찾기 힘들었다는 점이다.
그리고 한영사전 open api를 찾는다고 해도 사전 전체로부터 랜덤 단어를 뽑으면 고등학교 수준과 맞지 않은 영단어가 뽑혀 실용성이 없었다.
그래서 다른 방법을 찾던 중 웹 컴포넌트와 구글 시트를 연결해 데이터를 가져올 수 있다는 것을 알게 되었다.
최종적으로 수능 단어 1000개를 구글 시트로 불러와서 수정한 후 사용하게 되었다.
또 firebase에 텍스트 value를 저장할 때와 불러올 때 형식이 약간 다른 부분 때문에 학습 페이지에서 단어를 불러오는 과정에서 어려움을 겪었다.
firebase에서 value를 불러올 때의 형식의 규칙을 찾아 학습 페이지에서 활용하기 좋은 형식으로 바꾸어 해결했다.

최종 앱 사용방법
앱은 로그인, 계정, 학습, 세트 추가 화면까지 4개의 화면으로 이루어져 있다.
로그인 화면에는 오늘의 단어와 로그인 기능이 있다.
오늘의 단어 기능으로 앱을 열 때마다 수능 단어 1000개중에서 5개를 새로 뽑아 볼 수 있고, Firebase에 연결해서 만든 로그인 기능으로 회원가입/로그인을 할 수 있다.
로그인 하면 나오는 계정 화면에서는 아이디를 카톡으로 공유해 같이 학습할 친구를 추가하거나 비밀번호를 바꿀 수 있다.
세트 추가 화면에서는 친구들과 함께 학습할 카드 세트를 과목별로 저장할 수 있다.
학습 화면에서는 카드를 암기할 수 있는데, 카드를 알면 o, 틀리면 x를 선택함으로써 알때까지 카드를 학습할 수 있고, 사용자가 직접 뜻을 입력해 맞았는지 틀렸는지 확인할 수 있다.

practice 단어맞추기를 누르면 pop 소리가 제생이 되며 setcard2라는 함수가 실행된고 이 함수에 의해 랜덤으로 영어 단어가 뜬다. 빈칸에 해당 영어의 한국어를 쓰고 확인을 누르면 한국어와 ~~을 비교한 후 맞으면 right 소리가 나온 후 ‘정답입니다’ 알림이 뜨고 해당 영단어는 리스트에서 지운다. 틀렸다면 wrong 소리가 나온 후 ‘다시 시도해보세요’라는 알림이 뜨고 missed 리스트에 해당 영단어를 추가한다. 건너뛰기 버튼을 눌렀으면 missed 리스트에 해당 영단어를 넣고 타이머 호출? 만약 chosenlist에 남은 영단어가 없으면 놓친 단어가 뜬다. 놓친 단어가 없으면 ‘없음’이라고 표시된다. 돌아가기 버튼을 클릭한 경우 pop 소리와 함께 카드 선택 화면이 뜨고 영단어 카드 (name)을 선택할 수 있다. 선택한 영단어 카드를 firebase에서 태그로 찾는데 태그가 없을 경우 ‘error’가 표시된다. missed는 빈 리스트로 만들어준다. 태그를 받았을때 chosenlist에 값을 넣는다. ( ) 구분자를 통해 public이라는 단어가 있으면 모든 유저들에게 해당 영단어 카드가 기본 제공되어 학습할 수 있다. 카드 학습을 선택하면 영어 단어가 뜨고 o를 클릭하면 chosenlist에서 해당 영단어가 지워지고 right 소리가 난다. x를 클릭하면 이미 missed에 있는 영단어인지 확인하고 없으면 wrong소리와 함께 missed에 해당 영단어를 추가한다. x를 클릭한 영단어는 o를 클릭할 때까지 계속 뜬다. 만약 chosenlist에 있는 모든 영단어를 다 학습했으면 missed에 저장된 놓친 영단어가 표시된다. 놓친 영단어가 없으면 ‘없음’이라고 뜬다. addlist 추가할 카드 세트 이름과 단어와 뜻을 입력하고 저장을 누르면 단어를 localword에 뜻을 localmeaning 변수에 두고 FirebaseDB1의 카드세트이름 부분에 해당 영+한국어 단어를 리스트에 추가한다. 이후 right 소리가 나고 ‘저장되었습니다’ 알림이 뜨게한다. 느낀점 이서영: 앱을 만들며 앱인벤터의 새로운 기능을 여러가지 알 수 있어서 좋았다. 또 친구들에게 피드백을 받고 검토하는 과정에서 사용자 시점에서 앱을 바라보는 경험을 할 수 있었다. 결국 하나의 앱이나 프로그램이 상용화되기 위해서는 편리성이 중요하다는 점을 다시 느낄 수 있었다. 또한 오류들을 해결하는 과정에서 문제해결능력을 키울 수 있었다. 양민영: 친구가 전에 만들었던 앱 코딩을 이해하고 이를 다른 학생들도 이해하기 쉽도록 정리하여 웹 페이지에 추가하는 일을 맡았다. 다른 사람이 코딩한 것을 세세하게 분석하고 그것을 정리하는 것은 처음하는 경험이였다. 다른 사람이 작업한 코딩을 쉽고 빨리 이해하기 위해 또는 블럭 코딩 같은 경우 코딩에 순서가 없는데 이 순서를 더 빨리 파악하기 위해 앱 인벤터 블록 부분에 주석, 메모 기능이 있으면 더 좋았겠다고 느꼈다.
